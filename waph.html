WAPH - Web Application Programming and Hacking Course

In this course, I study basic web application development with front-end (HTML5, JavaScript, CSS) and back-end (PHP/MySQL). I explore web application vulnerabilities and attacks through hands-on exercises on the range, and apply secure programming principles and practices to prevent them.

So far, we have covered various Labs, Hands-on Exercises and Hackathons in this course shown below:

Labs Overview

Lab 0: Development Environment Setup

In this lab, I set up an Ubuntu Virtual Machine, installed the required software, and tested the Apache Web Server. I then initialized a Git repository, completed version control exercises, and generated SSH keys to establish a secure connection between my virtual machine and GitHub — getting comfortable with the full Git workflow in the process.

Lab 1: Front-end Web Development

In this lab, I explored the full request-response cycle between browser and server. I traced how an HTTP Request travels from the client to the Apache Web Server, gets processed by the CGI/PHP engine which generates a response, and is then packaged into an HTTP Response packet and sent back — where the browser finally renders it into a displayable webpage. I also used Wireshark to capture and visualize packets traveling between the client and server in real time.

Lab 2: Front-end Web Programming

In this lab, I built the overall structure of an HTML page and embedded local JavaScript to handle basic features on the client side without relying on the server for everything. I then used Ajax to send data to a web server endpoint (echo.php) and update the page dynamically without reloading. I simplified those Ajax calls using jQuery, and finally extended the page to communicate with external servers — fetching data from the Joke API and Agify API using the fetch() API.

Hackathons Overview

Hackathon 1: XSS Attack & Defense

The key lesson I took from this hackathon is that I can't defend something I don't know how to break. It had two tasks. In Task 1, I acted as a hacker — attempting to inject malicious scripts across increasingly hardened levels where the server applied word filters blocking the <script> tag. This pushed me to constantly think like an attacker and find creative ways around each filter. In Task 2, I switched roles and acted as the defender. I learned how output encoding using htmlentities() and htmlspecialchars() neutralizes all HTML entities automatically — eliminating the need to manually hunt for every possible bad input. This taught me how to reason about security guarantees: not just whether something works, but whether it holds no matter what a user throws at it.
